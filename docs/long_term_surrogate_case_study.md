# Long-Term Surrogate Case Study

This note is written for causal-inference users who think in terms of historical experiments, surrogate outcomes, and a new experiment whose long-term outcome is not yet available.

## Problem Setup

Suppose you run a new product experiment today and only observe short-term outcomes such as:

- clicks
- sessions
- conversion proxies
- early retention markers

You care about a long-term outcome such as long-run revenue or 12-week retention, but that outcome will only arrive later.

At the same time, you have historical experiments where you observed:

- surrogate outcomes `X`
- long-term outcomes `Y`
- experiment-arm assignments that changed the surrogate distribution

The paper's idea is to use the historical experiment arms as instruments to learn a surrogate-to-long-term map that is robust to surrogate-outcome confounding.

## Target Parameters

For one novel arm with surrogate data `X_new`, the target long-term mean is:

`E[h(X_new)]`

For a treated and control arm in a novel experiment, the long-term treatment effect is:

`E[h(X_new_treated)] - E[h(X_new_control)]`

Here `h` is the structural map from surrogates to long-term outcomes learned from historical experimental variation.

## Why IV Is Needed

If you regress long-term outcomes directly on surrogates in observational historical data, the surrogate-outcome relationship can be confounded:

- the same latent drivers may affect both the short-term surrogate and the long-term outcome
- a surrogate that looks predictive need not transport causally to a new intervention
- this is one way surrogate paradox problems appear in practice

Historical randomized experiments help because different experiment arms induce exogenous variation in surrogate outcomes.

## How The Package Maps The Problem To NPIV

- historical surrogate vector -> `X_hist`
- historical long-term outcome -> `Y_hist`
- historical experiment arms -> discrete instrument `Z`
- surrogate data from one novel arm -> `X_new`
- surrogate data from treated and control novel arms -> `X_new_treated`, `X_new_control`

The surrogate API wraps this mapping directly:

- `estimate_long_term_mean_from_surrogates`
- `estimate_long_term_effect_from_surrogates`

## Practical Interpretation Of "Past Experiments As Instruments"

In practice, each historical experiment arm is one source of exogenous perturbation to the surrogate distribution.

Examples:

- a ranking experiment arm
- a pricing experiment arm
- a messaging experiment arm
- a feature-rollout arm

When each unit is in one experiment arm, each arm becomes one discrete instrument level.

When units can be in multiple concurrent experiments, the instrument level is the full active set of experiment-arm assignments for that unit. This is discussed in `docs/experiment_encoding.md`.

## Support And Synthetic-Experiment Intuition

The package is most credible when the novel arm's surrogate distribution is reasonably represented by the historical experiment-arm variation.

Intuitively:

- the novel arm should not be far outside the surrogate patterns generated by historical experiments
- overlapping historical experiments are usable only if their encoded combinations have enough support
- if overlap produces many tiny cells, the problem is design support, not a nuisance implementation detail

That is why the package surfaces encoding diagnostics and refuses to silently merge sparse overlap combinations in v1.

## Minimal Workflow

```python
from discreteNPIV import estimate_long_term_effect_from_surrogates

effect = estimate_long_term_effect_from_surrogates(
    X_hist=X_hist,
    Y_hist=Y_hist,
    historical_arms=historical_arm_labels,
    historical_experiment_ids=historical_experiment_ids,
    X_new_treated=X_new_treated,
    X_new_control=X_new_control,
    n_splits=2,
    random_state=12,
)

print(effect.selected.estimate)
print(effect.selected.ci_lower, effect.selected.ci_upper)
```

## What The Output Means

- `effect.treated_mean.selected` is the estimated long-term mean for the novel treated arm
- `effect.control_mean.selected` is the estimated long-term mean for the novel control arm
- `effect.selected` is the estimated long-term treatment effect with a standard error and confidence interval
- `effect.encoding` shows how the historical experiments were encoded and whether some categories are sparse

## Assumptions In Plain Language

This package does not prove your causal design is valid, but it is built for settings where the paper's assumptions are a plausible approximation.

Key ideas:

- historical experiment arms provide exogenous shifts in surrogate outcomes
- historical long-term outcomes help identify the surrogate-to-long-term map
- the novel experiment contributes the target surrogate distribution
- the novel target should have enough support relative to historical experiment-induced surrogate variation

If these conditions are weak or implausible, the package can still run, but the causal interpretation becomes weaker.
